```mermaid
flowchart TD
    A([Начало]) --> B[Формирование пользовательского запроса<br/>на клиентской части]
    B --> C[Передача запроса на сервер]
    C --> D[Получение профиля пользователя<br/>и списка вещей из базы данных]
    D --> E[Формирование контекста подбора]
    E --> F{Все параметры контекста<br/>заданы?}
    F -- Нет --> G[Дополнение контекста:<br/>сезон, температура, погодные условия]
    F -- Да --> H[Разделение вещей по категориям]
    G --> H
    H --> I[Формирование кандидатных комбинаций]
    I --> J{Комбинация корректна?}
    J -- Нет --> K[Исключение комбинации]
    J -- Да --> L[Оценка комбинации по выбранным признакам]
    K --> M{Остались необработанные<br/>комбинации?}
    L --> N[Расчет итогового балла]
    N --> M
    M -- Да --> J
    M -- Нет --> O[Сортировка результатов]
    O --> P[Отбор ограниченного числа<br/>лучших вариантов]
    P --> Q[Формирование пояснения<br/>для каждого образа]
    Q --> R[Возврат результата пользователю]
    R --> S([Конец])
```

### Краткая подпись для диплома

Блок-схема отражает общий порядок работы проектируемого модуля подбора образов: от получения пользовательского запроса и данных гардероба до формирования допустимых комбинаций, их оценки, ранжирования и возврата пользователю ограниченного набора лучших результатов с пояснением.
